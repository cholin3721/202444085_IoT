TinkerCad(AutoDesk)라는 사이트에서  회원가입 후 아두이노 설계 및 코드 시뮬레이션을 돌릴 수 있다.
와이어는 전선을 의미, 사이트 내에서 블록코딩과 캐릭터 코딩이 있는데, 당연히 언어로 코딩(C/C++기반)

pinMode(a, b) : a라는 요소에 b라는 모드를 설정해놓음 이때 INPUT, OUTPUT 모드를 설정할 수 있는 데, 
INPUT과 OUTPUT의 기준은 아두이노, 즉, a라는 led 제어하는 모듈에 HIGH라는 신호를 보낸다 >> OUTPUT
외부 센서 연동으로 센서에서 받은 값을 아두이노에게 보낸다. >> INPUT 이라고 볼 수 있다.

아두이노가 가능한 것은 디지털 신호를 보내거나 받는 것, 아날로그 신호를 받는 것 까지 할 수있다.
아날로그 신호를 보내는 것은 억지로는 가능하나, 기본적으론 가능하지 않다.
그래서 아두이노를 보면 analogin 은 있지만 analog out 은 없고, 아두이노 코드에도 analogRead는 있어도 analogWrite는 존재하지 않는다.

아두이노 설명

gnd : 음극을 보내는 데 사용하는 포트, 주로 와이어 색은 검은색을 사용한다. 절대적인 것은 아니다.
디지털 포트 : 아두이노에서 처리하는 디지털 신호를 주고받는데 쓰는 포트
5V, 3.3V : 센서마다 필요로 하는 전압을 배출하는 포트, 오늘 한 초음파 센서(HC-SR04)는 5V를 사용해서 연결하였다.

회로 설명

저항 : 흘러나오는 전압에 비해, Led나 연결된 곳에 전압이 약할때, 그것을 통제해주는 역할. 저항이 커지면 전압이 작아지면서 출력이 약해지고, 저항이 약해지면 출력이 강해지지만, 한계 이상으로 강해지면 led에 필라멘트가 터지기도 한다. 단위는 옴

led : 불빛을 출력해주는 장치, 오늘은 초록색과 빨간색을 썼다. 필라멘트로 이루어진듯? 다리가 2개있는데 보통 긴쪽이 음극이라고 한다.

브래드보드 : 통칭 빵판, 같은 전합을 가로나 세로에 흘려서 쓰는 것으로, 아두이노에 센서 여러개를 연결할 때나 아두이노와 회로를 연결해주는 역할로 사실상 회로의 마더보드 같은 역할 같은 라인에 있는 것은 연결해도 되나, 보드 모델 별로 띄어져 있는 곳은 같은 극으로 연결되어 있지 않으니 주의 할 것,

와이어 : 전선, 메일과 피메일이 있다. 메일은 꽂는 역할, 피메일은 꽂을 수 있게 만든 전선이다. 주로 검은색은 음극, gnd, 빨간 색은 +극을 의미

HC-SR04 설명

사실상 교육용 초음파 센서, 보면 동그라미 두개가 있는데, 한 곳은 트리거, 한 곳은 에코, 트리거는 말 그대로 초음파를 발사하는 역할, 에코는 그 반사파를 받는 역할이다.

설명을 보면 작동 단계는

1. 10 마이크로 세컨드동안 trigger에게 명령을 전달해

2. 트리거에서 8사이클 의 초음파를 보낸 후

3. 에코는 그 반사파를 받아, 반사파 신호가 유지될때까지 High를 유지하다 신호가 오지않으면 Low로 바뀐다.

4. 즉 High였던 시간을 계산해 거리를 계산할 수 있는 셈(pulseIn(ECHO, HIGH) 함수)

5. 여기선 High 였던 시간 * 17 / 1000을 써서 cm단위로 분석하였다.

Week02 정리

TinkerCad(AutoDesk) 소개
TinkerCad는 아두이노 설계 및 코드 시뮬레이션을 돌릴 수 있는 사이트로, 회원가입 후 무료로 사용 가능하다.

전선은 **와이어(wire)**라고 부르며, 다양한 부품을 연결하는 역할을 한다.
사이트에서는 블록 코딩과 **캐릭터 코딩(C/C++ 기반의 텍스트 코딩)**을 지원하지만, 실제 아두이노에서는 C/C++ 기반 코딩이 필수다.
아두이노 코드 개념 정리
pinMode(a, b) : a라는 핀의 모드를 b(INPUT 또는 OUTPUT)로 설정하는 함수

OUTPUT: 아두이노가 **외부 모듈(예: LED)**에 신호를 보내는 경우
INPUT: 아두이노가 **외부 센서(예: 초음파 센서)**에서 신호를 받는 경우
아두이노가 처리 가능한 신호

디지털 신호(Digital Signal): HIGH(1) 또는 LOW(0) 값만 전송
아날로그 입력(Analog Input): analogRead(pin)으로 사용 가능
아날로그 출력(Analog Output): 직접적인 아날로그 출력은 불가능하지만, analogWrite(pin, value)을 사용해 PWM 방식으로 출력할 수 있음.
예: LED 밝기 조절, 모터 속도 조절 등
아두이노 포트 설명
GND(Ground) : 음극(-)을 보내는 포트, 일반적으로 검은색 와이어 사용
디지털 포트(Digital Pins) : 디지털 신호(0 또는 1)를 주고받는 핀
5V, 3.3V 전원 핀 : 각 센서에 필요한 전압을 공급하는 핀
예) 초음파 센서 HC-SR04는 5V 전압을 사용
회로 및 부품 설명
저항(Resistor)

전압을 조절하는 역할을 함.
저항이 크면 출력 전압이 약해지고, 저항이 작으면 출력 전압이 강해짐.
너무 낮은 저항을 사용하면 LED 필라멘트가 타버릴 수 있음.
단위: 옴(Ω)
LED(Light Emitting Diode)

빛을 출력하는 장치.
다리는 양극(긴 다리, +), **음극(짧은 다리, -)**로 구분됨.
전류의 방향을 주의해야 함.
브레드보드(Breadboard, 빵판)

같은 줄(세로 또는 가로)에 전압을 공유할 수 있는 보드
가로 방향(전력 레일): 같은 전압을 가짐
가운데 신호선(세로 방향): 일정 간격마다 단절되어 있음 → 같은 극으로 연결되지 않은 부분이 있으므로 주의!
센서 여러 개를 연결할 때 유용
와이어(Wire)

전선 역할, 메일(Male)과 피메일(Female) 와이어가 있음
메일(Male): 핀처럼 꽂을 수 있는 형태
피메일(Female): 꽂을 수 있는 구멍이 있는 형태
일반적인 색상 구분
검은색(GND): 음극(-)
빨간색(VCC, +): 양극(+)


초음파 센서(HC-SR04) 설명
초음파 센서의 역할

초음파를 발사하고 반사된 초음파를 받아 거리 측정
두 개의 동그란 모듈이 있는데, 하나는 Trigger(초음파 발사), 다른 하나는 Echo(반사 신호 수신)
작동 원리

trigger 핀에 10마이크로초(10µs) 동안 HIGH 신호를 줌
트리거에서 8사이클의 초음파 발사
초음파가 장애물에 부딪혀 반사되면 Echo 핀이 HIGH 상태 유지
반사 신호가 도착하면 HIGH에서 LOW로 변경
pulseIn(ECHO, HIGH)를 사용하여 HIGH였던 시간을 측정
거리 계산: 거리(cm) = (HIGH 유지 시간 * 17) / 1000


아두이노 IDE(통합 개발 환경) 사용법
아두이노를 처음 사용할 때, 포트와 보드가 Arduino Uno로 설정되어 있는지 확인해야 함.
코딩 후 업로드 버튼을 눌러야 아두이노 보드에 코드가 적용됨.


Serial 통신(컴퓨터-아두이노 간 통신)
Serial.begin(9600);

아두이노와 컴퓨터 간의 통신 속도를 설정
9600bps가 기본값이며, 값이 높을수록 데이터 전송 속도가 빠르지만 오류 발생 가능성이 커짐
Serial.print() vs Serial.println()

Serial.print() : 한 줄에 출력
Serial.println() : 출력 후 줄 바꿈(개행)


내장 LED(LED_BUILTIN) 설명
LED_BUILTIN은 아두이노 보드에 기본 내장된 LED를 의미함.
보드마다 다르지만 대부분 디지털 13번 핀에 연결된 LED를 제어하는 변수임.
외부 LED와 다르며, 물리적으로 겉에 보이지 않을 수도 있음.

Millisecond와 Microsecond의 관계
1 millisecond (ms) = 1,000 microseconds (µs)
delay(1000); → 1초 동안 대기
delayMicroseconds(1000); → 1ms 동안 대기
